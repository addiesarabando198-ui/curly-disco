## 2. Реализовано в этом цикле

### 2.1 B2B Billing Service (`b2b_api/services/billing_service.py`)

**Полный цикл рекуррентного биллинга через Tinkoff (T-Bank):**

| Метод | Описание |
|-------|----------|
| `create_card_binding_payment()` | Платёж 1₽ для привязки карты (`Recurrent=Y`, `CustomerKey=b2b_{client_id}`) |
| `save_rebill_id()` | Сохраняет RebillId после успешной привязки (вызывается из webhook) |
| `aggregate_monthly_usage()` | Агрегирует `b2b_checks` за месяц, считает по задачам |
| `create_billing_record()` | Создаёт/обновляет запись в `b2b_billing_summary` (upsert) |
| `charge_client()` | Init → Charge с RebillId, помечает invoice как paid/overdue |
| `bill_client()` | Полный цикл: агрегация → запись → списание |
| `bill_all_clients()` | Массовый биллинг всех активных платных клиентов |
| `get_billing_history()` | История биллинга клиента (до 36 месяцев) |
| `cancel_billing()` | Отключает автосписание, отменяет рекурент в Tinkoff |

**Ценовая модель:**

| Тариф | Цена/мес | Квота | Сверх квоты |
|-------|----------|-------|-------------|
| free | 0₽ | 100 | — |
| trial | 0₽ | 500 | — |
| basic | 2 900₽ | 2 000 | 5₽/проверка |
| standard | 9 900₽ | 10 000 | 5₽/проверка |
| premium | 29 900₽ | 50 000 | 5₽/проверка |
| enterprise | 99 900₽ | безлимит | — |

### 2.2 Billing Scheduler (`b2b_api/services/billing_scheduler.py`)

- Фоновая asyncio-задача, запускается при старте приложения
- Запускает биллинг 1-го числа каждого месяца в 10:00 MSK
- Биллит предыдущий месяц
- Интегрирован в `app.py` lifespan

### 2.3 Billing Admin Endpoints (`b2b_api/routes/admin.py`)

| Endpoint | Метод | Описание |
|----------|-------|----------|
| `/api/v1/admin/clients/{id}/billing/bind-card` | POST | Привязка карты (возвращает URL для оплаты) |
| `/api/v1/admin/clients/{id}/billing/charge` | POST | Ручное списание за указанный месяц |
| `/api/v1/admin/clients/{id}/billing` | DELETE | Отмена рекуррентных платежей |
| `/api/v1/admin/clients/{id}/billing/history` | GET | История биллинга (до 36 мес) |
| `/api/v1/admin/billing/run` | POST | Массовый запуск биллинга за месяц |

Все endpoints защищены `require_scope("admin")`.

### 2.4 Изменения в миграции (`b2b_api/migrations/b2b_tables.sql`)

Добавлены колонки в `b2b_clients`:
- `rebill_id TEXT` — RebillId от Tinkoff для рекуррентных списаний
- `billing_active INTEGER DEFAULT 0` — флаг активного автосписания
- `billing_order_id TEXT` — order_id последнего платежа привязки

## 4. Оставшиеся проблемы

### CRIT-1: SQLite не подходит для B2B production (без изменений)

**Файлы:** `core/config.py:20`, `core/db.py`, все сервисы

Вся система по-прежнему использует SQLite. При множественных B2B-клиентах:
- `BEGIN EXCLUSIVE` блокирует всю БД
- Нет connection pooling
- Горизонтальное масштабирование невозможно

**Решение:** Миграция на PostgreSQL + asyncpg + connection pool.

---

### CRIT-2: Отсутствие изоляции данных между тенантами (без изменений)

**Файлы:** `b2b_api/routes/check.py`, `b2b_api/migrations/b2b_tables.sql`

Все B2B-клиенты хранят данные в одних таблицах. Защита — только `WHERE client_id = ?`.

**Решение:** PostgreSQL RLS или schema-per-tenant.

---

### CRIT-3: In-memory состояние (без изменений)

| Компонент | Данные в памяти |
|-----------|-----------------|
| `SlidingWindowCounter` | Счетчики запросов/мин |
| `DailyCounter` | Счетчики запросов/день |
| `APILogger._queue` | Очередь логов (до 50 записей) |
| `APIKeyAuth._cache` | Кэш API-ключей (TTL 5 мин) |

При перезапуске rate limits сбрасываются — возможен abuse.

**Решение:** Redis для rate limiting и кэша.

---

### HIGH-5: Нет горизонтального масштабирования (без изменений)

Один процесс uvicorn, один файл SQLite, in-memory rate limiter.

**Решение:** PostgreSQL + Redis + Celery + gunicorn workers.

---

### HIGH-7: B2B API и teacher_mode — параллельные системы (без изменений)

Два мира квот, подписок и биллинга не связаны.

**Решение:** Единая система квот / абстракция над подписками.

---

### MED-2: White-label branding неполный (без изменений)

Нет привязки branding к B2B-клиенту.
